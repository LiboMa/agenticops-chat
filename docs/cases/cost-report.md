## Methodology

### Data Collection

Cost tracking uses three complementary sources:

1. **AWS CloudWatch Bedrock Metrics**
   - `aws/bedrock/InvocationInputTokens` — input tokens per model invocation
   - `aws/bedrock/InvocationOutputTokens` — output tokens per model invocation
   - Dimensions: `ModelId`, filtered by the three model tiers
   - Query period: aligned to scenario injection/completion timestamps

2. **Strands SDK Token Counters**
   - `AgentResult.metrics` provides per-invocation token counts
   - Aggregated in `TokenUsage` class (`cli/display.py`) for CLI sessions
   - Logged per SSE stream completion for web sessions

3. **Scenario Timing Files**
   - `.timing-results.csv` generated by each scenario's `inject.sh` and `verify.sh` scripts
   - Columns: `case_id, inject_ts, detect_ts, rca_ts, fix_ts, resolve_ts, input_tokens, output_tokens`
   - Aggregated by `infra/eks-lab/scenarios/run-all-scenarios.sh` into a summary report

### Cost Calculation Formula

```
cost = (input_tokens / 1000) * input_price + (output_tokens / 1000) * output_price
```

Where prices are determined by the model tier used for each agent in the pipeline.

### Comparison Methodology

The "Single Model (Opus)" baseline assumes all agents use Claude Opus 4.6 pricing. The "Tiered Models" column uses actual tiered pricing (Haiku for detect/scan/reporter, Sonnet for rca/sre/executor/main). This provides an apples-to-apples comparison showing the cost savings from model tiering alone.
